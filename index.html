Hexo教程

2019-03-12

很久以前就想搭一个技术博客了，但是一直拖延症病犯。最近因为找工作的原因，下定决心了要把这个博客搭起来。


找工作时一个很重要的方面是如何向面试官证明你的水平，比如说你英语很好，可以无障碍阅读英文技术文档，甚至翻译，但是要如何证明自己呢？如果你有一个博客在，平时翻译了一些文章在上面，简历上附上博客地址，面试官可以通过博客了解到真实水平，这将会大大降低面试时的聊天门槛。如果你的技术足够好，你写了很多有足够深度的技术文章，你甚至都不用去找工作，工作会自动来找你。

还有一个原因是处于互联网时代的人，一定要学会主动推销和展示自己，看过《未来是湿的》这本书的人就会理解，未来人与人之间的关系更多的是依存于互联网上的各种社区，而不是传统的大公司的组织结构，人与人的连接更直接而有效，而不必通过某个传统的组织来构成层级关系，在这个过程中，擅长在互联网营销展示自己的人可以获得巨大的流量红利，无论是对于找工作，还是创业，都会有巨大的帮助，而经营自己是一件长期的事情，冰冻三尺，绝非一日之寒，写博客是一个很好的开始。

写博客还有一个好处，它还会加深对知识的理解，内容消费和内容生产是完全不同的两个过程，在写博客做内容输出的过程中，你必定会对知识有更加深入完整的了解，如果一个知识点你能够用深入浅出的方式教会别人，你也就能够真正完全自信的说你掌握这个知识点了。


在网上比较了各种方案后，我选定了Hexo作为暂时写博客的框架，因为它简单易用，可以发布到 GitPage，也因为我对Node.js比较熟悉，后期比较好修改代码和加些自定义的内容。

第一篇博客不知道写些什么，就写个Hexo的教程吧，算是 Meta Blog :-D。

安装完成

Hexo 使用的是 Markdown 作为博客的写作语法，如果你不熟悉Markdown，可以看下这篇教程。

在这里给大家裂墙推荐一款Markdown编辑器： Typora。它是我用过的这么多 Markdown 编辑器之中最好用的，没有之一，它现在是免费测试阶段，你可以到它的官网下载。

Typora 有一个功能我认为是杀手级功能，就是它可以直接复制粘贴图片到文章中，这个功能实现起来并不复杂，但很多其它编辑器就是没有做到。

我在网上找到这个解决方案:

  1. 先用浏览器把这个binding文件下载下来
  2. 然后在命令行中执行 export SASS_BINARY_PATH=~/Downloads/darwin-x64-64_binding.node
  3. 这样再执行npm install hexo-render-scss --save就没问题了

主题安装完成之后重新用hexo s 来查看，就会发现主题已经变了。
